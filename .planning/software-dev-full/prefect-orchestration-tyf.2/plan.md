# Plan: prefect-orchestration-tyf.2 — Ship `~/.claude` context into worker pods

## Goal

Workers inside containers must see the same Claude Code context as the local
laptop user: `~/.claude/CLAUDE.md`, `prompts/`, `settings.json`, `skills/`,
`commands/`. Without these, slash commands (`/skill po`) won't resolve and the
agent loses the user-global instructions.

## Constraints discovered

| Source                  | Size  |
|-------------------------|-------|
| `~/.claude/CLAUDE.md`   | 32K   |
| `~/.claude/prompts/`    | 468K  |
| `~/.claude/commands/`   | 284K  |
| `~/.claude/skills/`     | 3.1M  |
| `~/.claude/settings.json` | 4K  |
| **total**               | ~3.9M |

**ConfigMap is not viable for the full payload**: a single ConfigMap is capped
at ~1MiB (etcd object size). The triage already flagged this. Any plan that
relies on `kubectl create configmap --from-file=$HOME/.claude` for the
unabridged tree will fail in apply.

Decision: **bake the full context into the worker image at build time** via a
`claude-context/` Docker build context, with a pre-build sync script that
populates it from the local `~/.claude` (whitelisted, sanitized). This is the
only delivery path that holds for skills (3.1M) without splitting into
multiple ConfigMaps or introducing an init-container/OCI-artifact flow we
don't need yet.

A small **ConfigMap overlay is still offered** for the *small* slice
(`CLAUDE.md` + `commands/` + sanitized `settings.json`, ~320K combined) so
those can be edited without an image rebuild. Skills/prompts stay baked.
This matches the issue's stated benefit ("update CLAUDE.md without rebuilding
images") for the file users actually iterate on.

## Affected files

- **New** `scripts/sync-claude-context.sh` — whitelisted copy from
  `$HOME/.claude/` into `claude-context/` (build-context dir, gitignored) +
  emits a sanitized `settings.json` (theme/model only, no hooks/MCP/tokens) +
  measures sizes + optionally writes a ConfigMap manifest for the small
  subset.
- **New** `claude-context/.gitkeep` — keep dir present even when empty so
  `Dockerfile` `COPY` always resolves; real content gitignored.
- **Modified** `.gitignore` — ignore `claude-context/*` except `.gitkeep`.
- **Modified** `Dockerfile` —
  - Add `FROM scratch AS claude-context` default stage (mirrors the existing
    `FROM scratch AS pack` pattern) so `COPY --from=claude-context` always
    resolves even when the build context is empty.
  - Add `COPY --from=claude-context --chown=coder:coder . /home/coder/.claude/`
    near the top of `runtime` (after `coder` user creation, before
    `entrypoint` install).
  - Caller invokes with `--build-context claude-context=./claude-context`.
- **Modified** `docker/entrypoint.sh` — currently overwrites
  `$HOME/.claude/settings.json` unconditionally. Change to *only* write the
  default settings.json when the file does not already exist (so a baked-in /
  ConfigMap-mounted settings.json wins). Same for `~/.claude.json`: keep
  unconditional (that file holds onboarding/trust state, not user context).
- **Modified** `k8s/po-worker-deployment.yaml` + `k8s/po-base-job-template.json`
  — add an *optional* projected/configmap volume mounting a
  `claude-context-overrides` ConfigMap at `/home/coder/.claude-overrides/`
  with a small init step in entrypoint that copies overrides on top of the
  baked tree (rsync-style overlay, only if mount exists). Keeps the override
  path opt-in; absent ConfigMap → image bake wins.
- **New** `k8s/claude-context-overrides.example.yaml` — example ConfigMap
  manifest for the small overrideable subset (CLAUDE.md, settings.json,
  commands/), generated by the sync script with `--emit-configmap`.
- **Modified** `README.md` (and/or `engdocs/work-pools.md` if more
  appropriate) — new section: "Shipping Claude context to workers".
  Documents:
  - Run `scripts/sync-claude-context.sh` before `docker build`.
  - What gets included / what's excluded (whitelist).
  - How to override CLAUDE.md/settings via ConfigMap without rebuilding.
  - Pod restart caveat after ConfigMap edit.
- **New** `tests/test_sync_claude_context.py` — unit-style test that runs
  the sync script against a fixture `~/.claude` (tmpdir), asserts whitelist,
  asserts secrets blacklist (`.credentials.json`, `projects/`, `cache/`,
  `secrets/`, `session-env/`, MCP tokens stripped from settings.json).
- **Modified** `scripts/smoke-compose.sh` (or new sibling) — verify
  `/home/coder/.claude/CLAUDE.md` exists and `/home/coder/.claude/skills/po/`
  resolves inside the container (`docker compose exec worker ls …`). Stops
  short of running `claude --print /skill po` because that needs OAuth/key
  and the smoke runs with `PO_BACKEND=stub`.

## Approach

1. **Sync script** (`scripts/sync-claude-context.sh`):
   - `set -euo pipefail`, default `SRC=${HOME}/.claude`, `DEST=./claude-context`.
   - Wipe `$DEST` (after a confirmation if non-empty and `--force` absent).
   - Whitelist copy: `CLAUDE.md`, `prompts/`, `skills/`, `commands/`. Use
     `rsync -a --delete` per directory.
   - Sanitize `settings.json` via `jq`: strip `hooks`, `mcpServers`, any
     `*token*`/`*key*`/`*secret*` keys, retain `theme`, `model`,
     `permissions` (if portable), `$schema`. Write to `$DEST/settings.json`.
   - Hard refusal list: error out if any of these exist under `$DEST` after
     copy — `.credentials.json`, `projects/`, `history.jsonl`, `cache/`,
     `secrets/`, `session-env/`, `ide/`, `paste-cache/`, `image-cache/`,
     `file-history/`, `backups/`, `archive/`, `plans/`, `.bkp`, `*.example.*`.
   - Print a size table; warn if `CLAUDE.md + commands/ + settings.json`
     exceeds ~900KiB (ConfigMap headroom).
   - `--emit-configmap PATH`: write a ConfigMap manifest containing the
     small subset to `PATH` (uses `kubectl create configmap … --dry-run=client
     -o yaml`).
   - Idempotent.
2. **Dockerfile** picks up the populated `claude-context/` via build-context.
   COPY happens after `coder` user exists so chown is correct. If caller
   forgets to run sync, the `FROM scratch AS claude-context` default makes
   COPY a no-op and the image still builds — pod will simply lack the
   context, exactly matching today's behavior, no regression.
3. **Entrypoint** stops clobbering settings.json when one is already
   present (image-baked or mounted). The `~/.claude.json` (onboarding state)
   stays unconditional — that file is environment-specific and must reflect
   container trust paths, not the laptop's.
4. **K8s overlay** is opt-in via a projected ConfigMap volume; entrypoint
   detects `/home/coder/.claude-overrides/` and `cp -rT` it onto
   `~/.claude/` before exec. Absent ConfigMap → noop.
5. **Project CLAUDE.md at /workspace**: out of scope for this bead; rig PVC
   wiring is tyf.4. We will document the expectation and add a smoke
   assertion that `/rig/CLAUDE.md` (where the rig actually mounts today,
   not `/workspace`) is reachable when the rig contains one. Avoid making
   tyf.4's PVC story a hard precondition.

## Acceptance criteria (verbatim)

> Pod has full ~/.claude context (CLAUDE.md, prompts, settings, skills, commands); skills resolve; sync script + docs ship

Expanded from the issue body:

1. Worker pod has `/home/coder/.claude/CLAUDE.md` + `prompts/` + `settings.json` + `skills/` + `commands/`.
2. `claude --print /skill po` inside the pod resolves without "unknown skill".
3. README documents how to populate the ConfigMap (or, after the size
   reality, the bake + optional ConfigMap-overlay flow) from local
   `~/.claude`.
4. A bootstrap script `scripts/sync-claude-context.sh` packages local
   `~/.claude` minus credentials/projects/cache into a manifest / build dir.
5. Project CLAUDE.md is reachable at `/workspace/CLAUDE.md` (gated on tyf.4;
   we verify the equivalent `/rig/CLAUDE.md` path that exists today).

## Test plan

- **Unit** — `tests/test_sync_claude_context.py` covering whitelist, blacklist,
  settings sanitization, idempotency, size warning emission, ConfigMap
  manifest emission shape.
- **E2E (compose smoke)** — `scripts/smoke-compose.sh` extended (or sibling
  `scripts/smoke-claude-context.sh`) that:
  - Runs sync against the developer's real `~/.claude`.
  - Builds the image.
  - `docker compose run --rm worker ls -la /home/coder/.claude/...` and
    asserts the five paths.
  - Asserts `claude --version` still runs (entrypoint not broken).
- **Manual** — documented `kubectl exec` step for AC 2, listed in README.
- **No Playwright** — backend infra task, no UI surface (triage `has_ui:
  false`).

## Risks

- **ConfigMap size hard limit**: addressed by baking into image; ConfigMap
  is overlay-only for the small subset. Sync script enforces the budget.
- **Secret leakage via settings.json**: jq sanitization is pivotal. Test
  the negative cases (token-bearing fixtures must be stripped). Whitelist,
  not blacklist, on top-level `~/.claude` to avoid future-file leakage.
- **Skill external-binary drift**: skills like `mgrep`, `firecrawl`, `po`
  reference binaries not present in the worker image. Not in scope —
  flagged for follow-up bead. Acceptance is about the markdown resolving,
  not the skill's tools running.
- **Entrypoint settings.json behavior change**: today it always overwrites.
  Switching to "only-if-missing" could surprise an operator who relied on
  the reset. Mitigation: document the change in entrypoint comments and the
  README; the previous behavior was unintentional clobbering.
- **Build-time bake ⇒ image-rebuild needed for CLAUDE.md edits**: ConfigMap
  overlay path mitigates for the small subset; full skill changes still
  require a rebuild. Documented.
- **Pod restart for ConfigMap pickup**: documented in README. Future
  improvement (reloader) not in scope.
- **Sync script must run on developer machines without exotic deps**:
  bash + rsync + jq + kubectl. All standard; check via `command -v` at
  script entry with friendly error messages.
- **`.gitignore` changes**: ensure no accidental commit of `claude-context/`
  (would leak the developer's local config into the repo). Pre-commit
  pattern is the gitignore + `.gitkeep` only.
- **Project CLAUDE.md / `/workspace`**: gated on tyf.4; do not block this
  bead on PVC wiring. Verify the rig-mount path that exists today.
